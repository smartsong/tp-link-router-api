> 2026-06-16 | 苏岚 @ 宋涛

## 目标

获取 TP-Link TL-R488GPM-AC 路由器的 REST API 访问能力，以便自动化管理设备。

---

## 第一步：登录接口定位

### 方法
通过浏览器 F12 → Network 面板抓包

### 发现

涛哥在浏览器中登录路由器后，抓到了登录请求：

```
POST http://192.168.0.1/
Content-Type: application/json; charset=UTF-8

{"method":"do","login":{"password":"<加密后的密码>"}}
```

响应：
```json
{"error_code":0,"stok":"0cf749710a8bca252a934e7bf3aa9b0a"}
```

**关键发现**：
- 登录接口就是根路径 `/`
- 返回 `stok` 作为后续 API 的认证 token
- 完全基于 token，无 cookie

### 算法还原

加密算法不是 RSA，而是**自定义 XOR + 查表**：

1. 两个硬编码字符串：
   - `input1` = `"RDpbLfCPsJZ7fiv"`（16字符）
   - `input3` = 长的查表字典（112字符）

2. 对 password 和 input1 逐字符进行 XOR运算
3. XOR结果对 input3 长度取模
4. 用取模结果作为索引查 input3 得到编码字符

**Python 实现**：
```python
KEY1 = "RDpbLfCPsJZ7fiv"
KEY2 = "yLwVl0zKqws7LgKPRQ84Mdt708T1qQ3Ha7xv3H7NyU84p21BriUWBU43odz3iP4rBL3cD02KZciXTysVXiV8ngg6vL48rPJyAUw0HurW20xqxv9aYb4M9wK1Ae0wlro510qXeU07kV57fQMc8L6aLgMLwygtc0F10a0Dg70TOoouyFhdysuRMO51yY5ZlOZZLEal1h0t9YQW0Ko7oBwmCAHoic4HYbUyVeU3sfQ1xtXcPcf1aT303wAQhv66qzW"

def encode_password(password):
    r = []
    g, m, f = len(password), len(KEY1), len(KEY2)
    for e in range(max(g, m)):
        t = l = 187  # 0xBB
        if e >= g:  t = ord(KEY1[e])
        elif e >= m: l = ord(password[e])
        else:        l, t = ord(password[e]), ord(KEY1[e])
        r.append(KEY2[(l ^ t) % f])
    return ''.join(r)
```

### 验证

用 Python 加密后的密码与浏览器抓包中的密码完全一致 ✅

## 第三步：API 调用规则

### 初始失败

用 `method: "do"` 尝试读取设备列表：
```json
POST /stok=xxx/ds
{"system":{"get_sys_info":null},"method":"do"}

→ error_code: -40209
```

### 搜索参考资料

搜索 CSDN，找到 2019 年的文章：
- **关键点**：读操作用 `method: "get"`，写操作用 `method: "do"`

修改后：
```json
POST /stok=xxx/ds
{"hosts_info":{"table":"host_info"},"method":"get"}

→ error_code: 0 ✅
→ 返回38台在线设备！
```

### 结论

| 操作类型 | method 值 | 示例 |
|---------|-----------|------|
| 读 | `"get"` | `{"hosts_info":{...},"method":"get"}` |
| 写 | `"do"` | `{"system":{"reboot":null},"method":"do"}` |

---

## 错误码含义

| 错误码 | 含义 |
|--------|------|
| 0 | 成功 |
| -40106 | 缺少 `method` 字段或认证格式错误 |
| -40209 | `method` 值错误（读操作用了 `"do"` 或 payload 格式错误） |
| -40401 | 未登录或 token 过期 |
| -54703 | URL 路径错误 |

## 附录：SU 框架源码分析

`su.js` 中包含 SU 前端框架，登录后加载菜单配置：

```javascript
(new a.su.Proxy({url:b,type:h,async:!1,autoLoad:!1})).read(c, ...);
```

Proxy 类负责与后端通信，但实际请求构建逻辑在压缩后的代码中。

菜单数据格式：
```json
{
  "name": "device_management",
  "text": "设备管理",
  "url": "device_management.htm"
}
```

每个菜单项对应一个 `.htm` 页面，页面中包含对应的 API 调用逻辑。
